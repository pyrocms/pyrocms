Coding Standards

This page describes the PyroCMS coding standards that all developers (core or module) should adhere to.

File Format

All files should be in UTF-8 format WITHOUT the BOM. You should always use Unix style line endings (LF). You can set Git to auto-fix the line endings.

Click here for more information

Tabs

All tabs should be real tabs, not spaces (a.k.a. “soft-tabs”). This allows each developer to choose his/her own tab size.

Code Indentation

Use Allman style indenting. With the exception of Class declarations, braces are always placed on a line by themselves, and indented at the same level as the control statement that “owns” them.

Correct

class Module_name {
    public function method_name()
    {
        foreach($array as $item)
        {
            // Code here
        }
    }
}

In-Correct

class Module_name {
    public function method_name() {
        foreach($array as $item) {
            // Code here
        }
    }
}

PHP Closing Tag

The closing tag at the end of a file should always be omitted. This ensures that no accidental output is sent before you want it to be.

Instead, mark the end of the file using the following comment style:

/* End of file filename.php */

Visibility

Always use visibility declarations on all class methods and variables.

Correct

class Module_name {
    public $property;
    protected $other_property;
    private $_ci;

    public function method_name()
    {
    }
}

In-Correct

class Module_name {
    var $property;
    var $other_property;
    var $_ci;

    function method_name()
    {
    }
}

Class and Method Naming

Class names should always have their first letter uppercase. Use the PHP 5 __construct() function for constructors unless absolutely necessary. Multiple words should be separated with an underscore, and not CamelCased. All other class methods should be entirely lowercased and named to clearly indicate their function.

Example

class Module_name {
    public function __construct()
    {
        // Construct Logic
    }

    public function method_name()
    {
        // Method Logic
    }
}

Variable Naming

All variable names should describe what they are without being overly verbose. They should be only contain letters, numbers and underscores (no CamelCase).

All private variables within a class should have their name begin with an underscore. This helps reduce confusion later in the code.

Correct

public $user_name;
public $password;
protected $bio;
private $_password_hash;

In-Correct

public $UserName;  // CamelCased
public $this_users_password;  // Too verbose
protected $b;  // Not Descriptive
private $password_hash;  // Missing the first underscore
