Translating PyroCMS

PyroCMS is a multilingual CMS. This means you can switch between different languages available to the system. If your language is not supported you can easily translate from one of the existing languages into your own and submit the change via a GitHub fork

Officially Supported

Sadly the PyroCMS development team only know how to speak English, so we can only "officially" support this translation. All that means is English is the only language we can be 100% things are accurate in, but the community translations are of a very good quality.

  • English

Community Supported

Community translations are submitted by the community and included in the core release of PyroCMS. They are checked for syntax errors and we make sure nothing obvious is wrong with them, but as mentioned above we cannot tell if they are 100% correct or not.

  • Chinese (Traditional)
  • Dutch
  • French
  • German
  • Polish
  • Portuguese (Brazilian)
  • Spanish

Adding new languages

To add a new language is pretty easy as long as you have experience working with Git. We have to use Git as otherwise it becomes almost impossible to merge work.

There are a few different files you will need to translate:

CodeIgniter language files

PyroCMS runs on top of CodeIgniter, so before you can translate PyroCMS you need to translate that too. Luckily that is not too much extra work.

system/codeigniter/language/<language-name>/

PyroCMS language files

system/pyrocms/language/<language-name>/
system/pyrocms/modules/<module-name>/language/<language-name>/

Translate module details

Modify the details.php file within each module directory to contain a name and description for your language. The language code should match the 2nd column on the IS0 639-2 Languages Codes page.

Install CKEditor language pack

Next you will need to download CKEditor (this is why WYSIWYG editor we use for the interface) and copy the language pack from lang/<language-code>.js to system/pyrocms/assets/js/ckeditor/lang/<language-code>.js. If you skip this step, the WYSIWYG will break on the Control Panel.

Enable language in config

Finally, add your language to system/pyrocms/config/language.php with ‘folder’ being the <language-name> folder used above. The name being the human readable name of the language. This step makes it available for users.

Testing languages

Languages will often be auto-detected based on the browser, but if you wish to change the language manually you can do this in your profile, in the Control Panel or manually by adding ?lang=<language-code> to any URL. E.g: http://example.com?lang=fr.

If you get any language related errors and want to switch back to English, just add ?lang=en to any URL.
